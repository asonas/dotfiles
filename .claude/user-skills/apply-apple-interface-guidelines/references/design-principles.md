# Design principles

Use a clear visual and semantic hierarchy. Keep the primary task prominent, secondary actions subordinate, and destructive actions distinguishable without relying on color alone.

Make controls and outcomes predictable. Reuse platform vocabulary, placement, and interaction patterns. Give immediate feedback for accepted input and visible progress for delayed work.

Design every meaningful state: initial, loading, empty, success, failure, offline, disabled, and interrupted. Preserve user input where possible and provide a recovery action near the failure.

Support multiple input and perception methods. Meaning must survive text enlargement, assistive technology, reduced motion, increased contrast, and color-vision differences.
